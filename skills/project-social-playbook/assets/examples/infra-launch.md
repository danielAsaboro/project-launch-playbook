# Worked Example — Infra Launch

**Project:** TraceDock  
**Archetype:** Workflow Wedge × Ecosystem Utility

## Core story angle

Solana teams do not need another dashboard. They need to understand failed transactions in minutes instead of burning half a day in logs.

## Example post

We built TraceDock because “instruction failed” is not a debugging workflow.

If your team ships on Solana, you already know the pattern:

- tx fails
- logs are noisy
- one engineer disappears for 3 hours
- everyone else waits

TraceDock turns that mess into a readable trace timeline so the fix starts faster.

If you want early access, reply with the nastiest debugging workflow your team still tolerates.
