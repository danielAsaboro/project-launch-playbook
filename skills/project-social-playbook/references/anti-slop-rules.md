# Anti-Slop Rules

Run this pass before returning the final 5 example posts.

## Kill these defaults

- “Excited to announce”
- “Thrilled to share”
- empty superlatives like “game-changing”
- generic “the future of”
- three abstract nouns in a row
- launch copy that never names a user, workflow, proof point, or objection

## Rewrite rules

- Replace abstract mission language with a user problem.
- Replace vague feature lists with one concrete wedge.
- Replace “join us” with a specific action.
- Replace inflated certainty with proof or honest framing.

## Style rules

- Emojis are optional, not a substitute for meaning.
- One em dash is fine when it improves clarity; repetitive em-dash rhythm is not.
- Short sentences beat slogan stacks.
- If a post could fit any startup, rewrite it.

## Quick checks

Ask:

1. Could this post belong to any crypto project?
2. Does it name a real user, workflow, or market problem?
3. Is the CTA concrete?
4. Does the post sound like a project account instead of a founder diary?

If any answer is weak, rewrite.
