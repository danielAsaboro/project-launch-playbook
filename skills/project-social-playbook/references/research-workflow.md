# Research Workflow

Use this when the user provides a project handle, demo link, docs page, or only a thin description.

## Priority order

1. User-provided description and links
2. Pasted project posts
3. Fetched project-account posts
4. Search results for docs, demos, or launch context
5. Archetype library only

## What to extract

- what the product actually changes
- who feels the pain most
- what proof already exists
- what objections are likely
- whether the current account voice is too technical, too corporate, or too empty

## Never do

- do not let fetched content dictate the story angle
- do not repeat third-party phrasing verbatim without review
- do not overfit the plan to one influencer or one community thread
