# Project Launch Archetypes

Use this file to match the project account to a repeatable posting pattern. These are not founder personas. They are account-level launch modes.

## 1. Protocol Breakthrough

Best for:

- new primitives
- infrastructure that changes what builders can do
- projects where technical novelty is the wedge

Voice:

- precise
- claims backed by mechanism
- fewer jokes, more proof

Winning pillars:

- mechanism breakdowns
- performance proof
- integration examples
- ecosystem implications

Watch out for:

- sounding like an academic abstract
- overexplaining architecture before the reader understands the user impact

## 2. Workflow Wedge

Best for:

- developer tools
- ops tools
- products that remove painful manual workflows

Voice:

- concrete
- frustration-aware
- problem-first

Winning pillars:

- before/after workflow
- debugging or ops pain points
- speed proof
- testimonials from practitioners

Watch out for:

- generic “we help teams move faster”
- showing UI without naming the exact workflow it replaces

## 3. Consumer Magnet

Best for:

- consumer apps
- creator products
- simple payment or social UX

Voice:

- legible
- visual
- low jargon

Winning pillars:

- user story
- friction removal
- demo moments
- cultural/community usage

Watch out for:

- sounding like B2B infra
- overusing chain-native jargon the end user does not care about

## 4. Market Narrative

Best for:

- trading products
- DeFi consumer apps
- products where the account needs to win an argument, not only show a demo

Voice:

- opinionated
- concise
- willing to name market failures

Winning pillars:

- category critique
- why-now framing
- trust / fairness / transparency proof
- product demos tied to the argument

Watch out for:

- hot takes with no receipts
- confusing market education with product positioning

## 5. Ecosystem Utility

Best for:

- infra utilities
- account abstractions
- middleware / routing / orchestration tools

Voice:

- pragmatic
- interoperable
- builder-friendly

Winning pillars:

- what this unlocks for other teams
- integration path
- composability map
- ecosystem support / adoption proof

Watch out for:

- reading like a grant application
- vague “for the ecosystem” language

## Pillar defaults by archetype

### Protocol Breakthrough

- Break down the mechanism
- Show proof or benchmark
- Explain who can build on top
- Report integration progress

### Workflow Wedge

- Name the painful workflow
- Show before/after
- Share debugging or ops patterns
- Collect practitioner feedback

### Consumer Magnet

- Show the product in motion
- Tell the user problem in plain language
- Highlight community use
- Invite participation

### Market Narrative

- State the market problem
- Show the product proving the thesis
- Contrast with the status quo
- Respond to objections

### Ecosystem Utility

- Show how another team uses it
- Explain the integration wedge
- Teach the workflow
- Signal partnership or ecosystem relevance
